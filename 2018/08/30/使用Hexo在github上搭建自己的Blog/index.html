使用Hexo在github上搭建自己的Blog

Posted on 2018-08-30 | Edited on 2018-08-31 | In 博客搭建

最近项目比较闲，都在做一些翻译的工作，沉淀下来的东西多了，就想着搭建自己的Blog。但是从零开始搭建，包括服务器搭建，容器，域名，前端工作等等这些计算下来，成本比较高。于是乎，所以博主决定先直接在github.io上使用一些现有的博客框架创建自己的blog。

在网上找了一些，好多关于HEXO的推荐，去官网上看了看，文档和源码都一应俱全，决定就在此基础上去搭建自己的博客啦。

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

准备工作

1. 本地环境

由于 HEXO 是基于 NodeJs 的，所以需要准备node环境，最后本地生成的静态文件我们需要挂在 Github 上，因此还需要安装 git。下面列举一下需要提前准备的环境。

  • node
  • npm
  • git
  • Hexo

npm会在node里一并安装，关于node，git的安装我就赘述，不清楚的读者请自行Google。下面说一下 HEXO 的安装，

2. Git仓库

由于需要将本地生成的静态包上传到Git的服务器上，git提供给每一个注册用户一个特有的工作域，为username.github.io，下面就简要讲解一下如果准备这一工作域。
首先，在有 GitHub 账号的前提下，创建仓库，命名为username.github.io，这里username为你的 GitHub 的用户名，注意该repo的命名必须为该形式。到此为止，git 环境就准备好了。

发布到GitHub

经过了上述步骤，我们在本地上建立了自己的文章，到此本地的配置都已经 ready 了，接下来我们需要做的是将本地的静态文件发布部署到自己的 github.io 上。

在部署之前，我们需要先安装hexo-deployer-git自动部署发布工具，

其中

  • hexo clean: 清除缓存文件db.json和已生成的静态文件 public。在某些情况（尤其是更换主题后），如果发现对站点的更改无论如何也不生效，就可能需要运行该命令。
  • hexo g: g也可以是generate，即生成静态文件。
  • hexo d: d也就是deploy，即为发布部署生成的静态文件。发布的位置我们已经在前面_config.yml文件中对deploy参数进行了设置，也就是我们自己的 github.io 。

命令运行完成后，我们就可以直接去https://username.github.io/验证是否成功，要是与本地服务相同，那么恭喜你成功搭建了自己的博客平台啦。
