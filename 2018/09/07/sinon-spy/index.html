Spy

官方说明

A test spy is a function that records arguments, return value, the value of this and exception thrown (if any) for all its calls. There are two types of spies: Some are anonymous functions, while others wrap methods that already exist in the system under test.

测试中的通过 spy 生成一个间谍函数，这个函数会记录对应函数调用的参数，返回值，this值以及抛出的异常。spy 有两种使用场景，一种是是直接生成一个匿名间谍函数；另一种是对已有的函数进行封装并进行监听。

场景一: 通过 spy 生成匿名间谍函数，一般以参数的形式传入到需要调用的方法中，不会运行被替换的真实函数。常常用于判定回调函数是否执行。
场景二: 通过对已有的函数进行封装，调用该函数时，对其进行监听，会运行该函数中的内容。

构造方法
3 种创建 spies 的方法: sinon.spy()

  • var spy = sinon.spy();
    创建一个匿名函数用于记录其每次被调用的传入参数、this 值、异常和返回值。
  • var spy = sinon.spy(myFunc);
    监视一个已有的函数。
  • var spy = sinon.spy(object, "method");
    为 object.method 创建一个 spy，即使用 spy 替换原方法。这个 spy 的表现在各方面都和原方法很像。原方法可以通过调用 object.method.restore() 方法还原。返回的 spy 就是替换原方法的函数对象，即 spy === object.method。

卸载

* `spy.restore`只能是基于已有函数的，验证只有通过`sinon.spy(object, "method")`这种初始化方式的 spy 函数能卸载。

2. 场景二：封装函数

官网原文：

Creates a spy for object.method and replaces the original method with the spy.

总结

总的来说，spy 主要是用于监听函数的调用状态，包括调用次数，参数，返回值等等，会真实调用被监听的函数。
