Stub

Stub的官方介绍：

Test stubs are functions (spies) with pre-programmed behavior.
They support the full test spy API in addition to methods which can be used to alter the stub’s behavior.
As spies, stubs can be either anonymous, or wrap existing functions. When wrapping an existing function with a stub, the original function is not called.

翻译：stubs 是具有预编程行为的函数，他们不仅支持 spy 的所有 API，还能操作目标函数的行为（可以理解为彻底替换目标函数），换句话说，当你使用spy，原始的函数依然会被调用，但如果使用stub，原始函数就不会被执行了。
同 spy 相同，他既可以为匿名函数，也能去封装监听已有的函数。与 spy 不同的是，当封装监听已有的函数时，原函数并不会被调用。

对于 stub 的使用场景：

  • 希望阻止一个方法被直接调用，如网络请求等耗时操作，或其它测试变复杂或慢外部函数，使用 stub 对其进行替换，避免测试中的耗时操作；
  • 在测试中控制一个函数的行为，以强制代码沿特定路径执行，使用 stub 替换该函数的输出，进而根据函数的响应来触发不同的代码流程；
  • 测试不寻常的条件，如抛出异常，使用 stub 抛出异常；

代理返回值和抛出异常

stub 不仅能代理函数调用，还能代理函数调用后的返回值和异常的抛出，这一点能为我们验证内部逻辑提供有力的支持。
